### Вопрос 1. Что является сущностью?

Это существительное, обозначающее систему. Если это человек, объект или событие, о котором мы собираемся записывать данные, это сущность.
В мире IT-инфраструктуры Server (Сервер) — это центр всего. Он — та основа, на которой работают сайты, базы данных и приложения. Именно поэтому я выбралa его в качестве своей главной сущности; без серверов сеть просто не существует. Это фундамент, от которого зависит работа всех остальных сервисов.

### Вопрос 2. Какие у него атрибуты?

| Атрибут | Тип | Описание |
| :--- | :--- | :--- |
| name | str | Имя сервера (например, "Web-Server") |
| priority| int | Уровень приоритета сервера (от 1 до 10) |
| connections | int | Текущее количество подключений (не может быть отрицательным) |
| status | str | Логическое состояние: выключен, активен, ошибка |

Также добавлен атрибут класса MAX_CONNECTIONS = 5000 — единый лимит максимального числа подключений для всех экземпляров серверов. 
     
#### Поведение (методы):

*   turn_on() — включает сервер, переводя его в статус активен.
*   crash() — имитирует критический сбой, автоматически сбрасывает все подключения до 0 и меняет статус на ошибкa.
*   add_connections(count) — добавляет новые подключения (работает только если сервер активен).

### Вопрос 3. Какие инварианты?

*   name не может быть пустой строкой.
*   priority находится в диапазоне от 1 до 10 включительно.
*   connections не может быть отрицательным числом.
*   Количество подключений не может превысить MAX_CONNECTIONS (если попытаться задать больше, оно упрется в потолок 5000).
*   status принимает только значения из множества {"выключен", "активен", "ошибка"}.
*   Нельзя изменять количество подключений, если сервер находится в состоянии выключен или ошибка.
*   Все базовые проверки вынесены в отдельный файл validate.py и проверяются как в конструкторе, так и в сеттере.

### Вопрос 4. Что значит «равенство»?

Когда можно сказать, что два сервера — это одно и то же?

Если я просто позволю Python самому решать, он посчитает два сервера разными просто потому, что они были созданы в разное время в памяти. Но для моей логики инфраструктуры это не подходит. Я решилa, что истинным идентификатором сервера является его имя и приоритет в сети.

Другими словами, если у меня есть два объекта с именем "Web-Server", и у обоих приоритет 5, логически это один и тот же сервер (или один и тот же кластер), даже если в данный момент на одном висит 100 подключений, а на другом 0. Поэтому в коде я сравниваю только эти два параметра. Если они совпадают, программа считает их "равными".

### Вопрос 5. Есть ли состояние?

Да. Объект Server имеет три чётко выраженных состояния:

| Состояние | Описание | Ограничения |
| :--- | :--- | :--- |
| выключен | Сервер остановлен (по умолчанию) | Нельзя добавлять подключения или менять их количество. |
| активен | Нормальная работа | Доступны все действия, сервер принимает подключения. |
| ошибка | Произошел сбой | Подключения падают до нуля, любые новые подключения блокируются. |

Переходы между состояниями:
*   выключен -> активен: через метод turn_on()
*   активен -> ошибка: через метод crash()
